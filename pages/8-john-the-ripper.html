Lesson 8 · TryHackMe: John the Ripper — The Basics

John the Ripper

John (specifically Jumbo John) is a fast, versatile dictionary-attack cracker. You can't reverse a hash, but you can hash millions of guesses and compare. This lesson walks the whole room: raw hashes, NTLM, /etc/shadow, single-crack mode, custom rules, and cracking Zip / RAR / SSH keys.

Jumbo John rockyou.txt unshadow zip2john · rar2john · ssh2john Simulated john below
Legal & safe note The terminal below simulates John for this room's known hashes so you can learn the workflow offline. Only crack hashes you're authorised to (CTFs, your own systems, engagements).

Why cracking works at all

Hashing is a one-way (P) operation: easy to compute, infeasible to invert (NP). But if you know the algorithm, you can hash a huge dictionary of candidate passwords and look for a match. That's a dictionary attack — John's core trick.

TermMeaning
Jumbo JohnThe community edition with extra formats and the *2john tools.
WordlistList of candidate passwords. This room uses rockyou.txt (2009 breach).
FormatWhich hash algorithm John should assume, e.g. raw-md5, nt, sha512crypt.

On the AttackBox / Kali it's pre-installed — check with john (look for 1.9.0-jumbo-1). rockyou lives at /usr/share/wordlists/rockyou.txt.

Basic syntax

john [options] [file path]
FormWhat it does
john --wordlist=<list> <file>Wordlist mode, John auto-detects the format.
john --format=<fmt> --wordlist=<list> <file>Force a specific format when auto-detect misfires.
john --show <file>Re-print already-cracked results (stored in ~/.john/john.pot).

Format prefix trap: for a plain hash you usually prefix raw- (e.g. raw-md5). Check with john --list=formats | grep -iF "md5".

Identifying a hash first

If auto-detect fails, identify the type, then set --format. Use an online identifier or the Python hash-identifier tool (hash-id.py):

wget https://gitlab.com/kalilinux/packages/hash-identifier/-/raw/kali/master/hash-id.py
python3 hash-id.py

Feed it a hash and it lists probable formats — try them top-down.

Task 5 — Windows NTLM / NThash

NThash (loosely "NTLM") is how modern Windows stores passwords in the SAM database, or in Active Directory's NTDS.dit. You'd dump it with a tool like Mimikatz. Sometimes you "pass the hash" instead of cracking — but weak policies make cracking viable.

john --format=nt --wordlist=/usr/share/wordlists/rockyou.txt ntlm.txt
Show cracked answer (Task 5)

ntlm.txt (--format=nt) → mushroom

Task 6 — /etc/shadow with unshadow

Linux hashes live in /etc/shadow (root-only). John needs it combined with /etc/passwd — that's what unshadow does:

unshadow local_passwd local_shadow > unshadowed.txt
john --wordlist=/usr/share/wordlists/rockyou.txt unshadowed.txt
# if needed:  --format=sha512crypt

You can use whole files or just the relevant line from each. The root line here is a $6$ hash → sha512crypt. Crack it on the VM (salted, so not in the offline sim). Files in ~/John-the-Ripper-The-Basics/Task06/.

Task 7 — Single crack mode & word mangling

Instead of a wordlist, --single derives guesses from the username by mangling it: Markus → Markus1, MArkus, Markus!… It also reads the GECOS field (full name, etc.) from passwd/shadow entries.

The file must be in username:hash form so John knows the base word:

# file joker.txt:  joker:7bf6d9bb82bed1302f331fc6b816aada
john --single --format=raw-md5 joker.txt
Show cracked answer (Task 7)

User joker → Jok3r (single-crack mangles "joker").

Task 8 — Custom rules

Password policies push users into predictable patterns: capital first, number + symbol at the end — Polopassword1!. Custom rules in john.conf (/opt/john/john.conf on the AttackBox) mutate a wordlist to match that shape.

[List.Rules:PoloPassword]
cAz"[0-9] [!$%@]"
TokenEffect
cCapitalise the first letter.
Az"…"Append the characters/sets that follow.
A0"…"Prepend the characters/sets that follow.
[0-9]Any digit; [A-Z] upper, [a-z] lower, [!$%@] those symbols.

So polopassword → Polopassword1!. Invoke it with:

john --wordlist=/usr/share/wordlists/rockyou.txt --rule=PoloPassword hash.txt

Stuck? Jumbo John ships tons of rules — peek around line 678 of john.conf.

Tasks 9–11 — Archives & SSH keys (the *2john tools)

The pattern is always the same: a helper converts the file into a hash John understands, then you crack it with a normal wordlist run.

TargetConvertCrack
Zip (Task 9)zip2john secure.zip > zip_hash.txtjohn --wordlist=…/rockyou.txt zip_hash.txt
RAR (Task 10)rar2john secure.rar > rar_hash.txtjohn --wordlist=…/rockyou.txt rar_hash.txt
SSH key (Task 11)ssh2john id_rsa > ssh_hash.txtjohn --wordlist=…/rockyou.txt ssh_hash.txt

No ssh2john binary? Use python3 /opt/john/ssh2john.py (AttackBox) or python /usr/share/john/ssh2john.py (Kali). These need the real files + rockyou — do them on the lab VM.
