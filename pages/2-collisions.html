Lesson 2

Hash collisions

A collision is when two different inputs produce the same output. Good hash functions make this astronomically unlikely — bad ones don't.

Why collisions must exist: the pigeonhole principle

Inputs are unlimited in size, but outputs are a fixed number of bits. More pigeons than pigeonholes means some hole holds two pigeons. With n bits of output there are only 2ⁿ possible digests:

Output bitsPossible values (2ⁿ)
416
8256
1665,536
128 (MD5)≈ 3.4 × 10³⁸

So collisions are unavoidable in theory. A good hash function just makes the probability negligible and makes it infeasible to engineer one on purpose.

MD5 and SHA1 are broken Researchers can now engineer collisions in both. MD5 collisions are trivial today; SHA1 fell to the Shattered attack (2017). Do not use either to hash passwords or verify integrity against a motivated attacker.

Collision vs. reversing — don't mix them up

TermMeaning
CollisionFind any two inputs with the same hash.
Second-preimageGiven one input, find a different input with the same hash.
PreimageGiven only a hash, find an input that produces it (this is "reversing").
