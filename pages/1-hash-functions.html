Lesson 1

What is a hash function?

A hash function eats any amount of data and spits out a short, fixed-size fingerprint. It only goes one way.

The plain-English version

A hash function takes input of any size and produces a fixed-size digest (the output). Four properties matter:

  • Fixed output size. "T" and a 4 GB movie both hash to the same length.
  • Deterministic. Same input → same output, every single time.
  • One-way. Easy to compute forward, impractical to reverse. There is no key — this is the big difference from encryption.
  • Avalanche effect. Change one bit of input → the whole output changes.
Hashing ≠ encryption Encryption is reversible with a key. Hashing has no key and is not meant to be reversed at all. If someone says "decrypt this hash", they mean crack it (Lesson 5).

See the avalanche effect

The letters T (hex 54, binary 01010100) and U (hex 55, binary 01010101) differ by a single bit. Type into the box — flip that one character and watch every digest change completely.

Output sizes & encoding

Raw hash output is bytes, usually printed as hexadecimal (2 hex chars per byte). That's why lengths are predictable:

AlgorithmOutput bitsOutput bytesHex length
MD51281632
SHA11602040
SHA-2562563264
SHA-51251264128
Handy math bytes = bits ÷ 8. Hex length = bytes × 2. So MD5 = 128 ÷ 8 = 16 bytes = 32 hex chars.

Prove it in the terminal

Files file1.txt (contains T) and file2.txt (contains U) are loaded. Try these:

hexdump -C file1.txt
md5sum file1.txt
md5sum file2.txt
sha256sum file1.txt
What you should notice A one-bit input change produces totally different MD5, SHA1, and SHA256 outputs — yet the same input always gives the same hash.
