Common mistakes & problem-solving tactics

The traps people fall into, a repeatable workflow for unknown hashes, and the external tools worth bookmarking.

Common mistakes

1. Confusing hashing, encoding, and encryption "Decode this MD5" makes no sense — MD5 is a one-way hash, not reversible. Base64 is not encryption. Encryption needs a key. Get the category right first.
2. Identifying a hash by length alone 32 hex chars could be MD5, NTLM, or MD4. 64 could be SHA-256 or SHA3-256. Use the source/context: web-app DB → MD5; Windows SAM → NTLM.
3. Trusting automated hash-ID tools blindly hashID/hash-identifier are reliable on prefixed hashes ($6$, $2b$) but routinely mislabel plain hex hashes. Treat their output as a hint, not truth.
4. Forgetting the salt Salted hashes ($6$…, bcrypt) can't be cracked with a plain rainbow table. hashcat needs the whole $prefix$salt$hash string, not just the hash portion.
5. Using MD5/SHA1 (or fast hashes) for passwords They're fast and broken. Passwords need slow, salted KDFs: Argon2, scrypt, bcrypt, PBKDF2.
6. Wrong hashcat mode / wrong pass-vs-salt variant md5($pass.$salt) (10) ≠ md5($salt.$pass) (20). HMAC-SHA512 key=$pass (1750) ≠ key=$salt (1720). Read the mode name exactly.
7. Cracking in a VM and expecting GPU speed VMs usually can't touch the host GPU. Run hashcat on the host; John on CPU is fine in a VM.

Tactics: cracking an unknown hash

Repeatable workflow
  1. Prefix? $…$ → look it up (reliable). Done identifying.
  2. No prefix? Count hex length → shortlist candidates.
  3. Context decides. Where did it come from? App DB, SAM, shadow, network capture?
  4. Pick the hashcat mode (or let John auto-detect).
  5. Straight attack first: -a 0 with rockyou.txt.
  6. Still nothing? Add rules (-r best64.rule), then masks (-a 3), then bigger wordlists.
  7. Salted & slow (bcrypt/$6$)? Expect it to be slow — keep the wordlist tight.

External tools & references

  • CrackStation ↗ Free rainbow-table lookup for unsalted MD5/SHA1/NTLM.
  • Hashes.com ↗ Online lookup + hash identifier.
  • Hashcat example hashes ↗ Authoritative mode-number ↔ hash-format table.
  • John the Ripper ↗ CPU cracker, great auto-detection, works in VMs.
  • Shattered ↗ The SHA1 collision attack.
  • SecLists ↗ Wordlists beyond rockyou.txt.
  • man 5 crypt · man 5 shadow Local docs for Unix hash prefixes and the shadow file format.
