Lesson 4

Recognising a hash

Before you can crack a hash you need its type. Prefixes are reliable; raw hex length is a guess. Automated tools mix things up — context wins.

Prefixes: the reliable signal

Unix/Linux password hashes in /etc/shadow use the format $prefix$options$salt$hash. The prefix names the algorithm:

PrefixAlgorithmStrength
$y$yescryptModern default, strong
$gy$gost-yescryptStrong
$7$scryptStrong
$2b$ $2y$ $2a$ $2x$bcryptStrong
$6$sha512cryptOK (older Linux)
$md5SunMD5Weak
$1$md5cryptWeak

Reading a shadow line

strategos:$y$j9T$76UzfgEM5PnymhQ7TlJey1$/OOSg64dhfF.TigVPdzqiFang6uZA4QA1pzzegKdVm4:19965:...
           │   │   └ salt                └ hash
           │   └ options (parameter)
           └ $y$ = yescrypt
yescrypt hash size yescrypt's hash portion is 256 bits. The prefix $y$ is your dead giveaway.

Windows: NTLM looks just like MD4/MD5

Windows stores password hashes in the SAM using NTLM (a variant of MD4). NTLM is 32 hex chars — visually identical to MD5 and MD4. Only context tells them apart (a Windows SAM dump → NTLM; a web-app DB → probably MD5).
